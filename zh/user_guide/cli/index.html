    CLI 参考

    qps（QPhase Shell）命令行界面是与 QPhase 框架交互的主要工具。它用于项目初始化、任务执行、插件管理和配置生成。

    全局选项

    所有命令都支持以下全局标志：

    • --help：显示帮助信息并退出。
    • --version：显示已安装的版本号。
    • --verbose / -v：启用详细调试日志。
    • --log-file PATH：将日志重定向到指定文件。

    项目管理

    qps init

    在当前目录初始化一个新的 QPhase 项目。

    qps init

    功能： 1. 创建标准目录结构： * configs/：配置文件。 * plugins/：本地用户插件目录。 * runs/：仿真结果输出目录。 2. 生成默认的 configs/global.yaml 文件。

    仿真执行

    qps run

    执行在 configs/jobs/ 目录中定义的仿真任务。

    qps run [JOB_NAME] [OPTIONS]
    • 参数：
      • JOB_NAME：位于 configs/jobs/ 中的任务配置文件名（不含扩展名）。
    • 选项：
      • --list：列出所有可用的任务配置并退出。
      • --verbose / -v：启用详细调试日志。

    示例： 

    # 执行单个任务
qps run vdp_sde

# 列出可用任务
qps run --list

# 使用详细日志运行
qps run --verbose vdp_sde

    插件管理

    qps list

    列出当前环境中所有已注册的插件。

    qps list [CATEGORIES]
    • 参数：
      • CATEGORIES（可选）：以逗号分隔的命名空间列表进行过滤（例如 backend、model）。使用 . 列出所有类别。

    示例： 

    qps list backend
# 输出：
# Available Plugins
# backend: (2 plugins)
#   numpy  (qphase.backend.numpy)
#   torch  (qphase.backend.torch)

    qps show

    显示特定插件的详细信息，包括其描述、源代码位置和配置模式。

    qps show [PLUGIN_ID]
    • 参数：
      • PLUGIN_ID：插件的完整标识符（例如 backend.numpy 或 model.kerr_cavity）。

    qps template

    为特定插件生成配置模板。这对于快速创建新配置文件很有用。

    qps template [PLUGIN_ID]
    • 参数：
      • PLUGIN_ID：插件的完整标识符。

    示例： 

    qps template model.kerr_cavity
# 输出：
# # Configuration for model.kerr_cavity
# chi: 1.0  # Nonlinearity parameter
# ...
