    快速开始指南

    本指南将帮助您设置 QPhase 并运行您的第一个仿真。

    1. 安装

    我们强烈建议使用虚拟环境来保持项目依赖的整洁。

    步骤 1：创建虚拟环境

    打开终端（PowerShell 或 Bash）并运行：

    # 创建名为 '.venv' 的虚拟环境
python -m venv .venv

# 激活它
# Windows (PowerShell):
.venv\Scripts\Activate.ps1
# Linux/macOS:
source .venv/bin/activate

    步骤 2：安装 QPhase

    使用 pip 安装包：

    pip install qphase

    2. 初始化项目

    当您有一个专门的仿真文件夹时，QPhase 工作得最好。这可以使您的配置文件和结果保持有序。

    # 为您的研究项目创建一个文件夹
mkdir my_research
cd my_research

# 初始化 QPhase 结构
qps init

    这将创建以下文件夹： * configs/：您告诉 QPhase 要运行什么的地方。 * plugins/：您放置自定义物理代码的地方。 * runs/：QPhase 保存数据的地方。

    3. 创建您的第一个任务

    "任务"是单次仿真运行。您在 YAML 文件中定义它。 创建一个名为 configs/jobs/test_run.yaml 的新文件并粘贴以下内容：

    # configs/jobs/test_run.yaml
name: test_run

# 1. 选择引擎（SDE 求解器）
engine:
  sde:
    t_end: 10.0
    dt: 0.01
    n_traj: 100

# 2. 选择物理模型
# （这里我们使用内置的示例模型（如果可用），或者您可以定义自己的模型）
# 对于此示例，假设我们有一个 'vdp_oscillator' 模型可用。
# 如果没有，您可能需要先编写一个插件（请参阅开发者指南）。
plugins:
  model:
    vdp_two_mode:  # 这是一个内置的示例模型
      D: 1.0       # 扩散强度

  backend:
    numpy:         # 在 CPU 上运行
      float_dtype: float64

    4. 运行仿真

    现在，告诉 QPhase 运行您刚刚定义的任务：

    qps run test_run

    您应该看到一个进度条： 

    [INFO] Loading 1 configuration file(s)
[INFO] Starting job execution
[test_run] 100.0% ~00:00
[INFO] All 1 jobs completed successfully

    5. 检查结果

    查看 runs/ 文件夹。您将看到一个带有时间戳的新目录：

    runs/
└── 2025-12-29T12-00-00Z_test_run/
    ├── config_snapshot.json  # 使用的确切设置
    └── results.h5            # 仿真数据（格式取决于引擎）

    恭喜！ 您已成功运行了一个可复现的物理仿真，而无需编写任何样板代码。
