    配置指南

    QPhase 使用基于 YAML 的分层配置系统，旨在确保可复现性、灵活性和易用性。

    配置方案

    QPhase 支持两种配置方案。我们强烈推荐使用扁平方案，因为它更清晰、更易于组合。

    1. 扁平方案（推荐）

    在此方案中，一个文件代表一个任务。这避免了深层缩进，并使得在不同管道中重用任务变得容易。

    文件： configs/jobs/my_simulation.yaml 

    name: my_simulation

# 引擎配置
# 键（例如 'sde'）决定引擎类型。
engine:
  sde:
    dt: 0.01
    t_end: 100.0
    n_traj: 1000

# 插件配置（顶层键）
# 键必须匹配已注册的插件类型（例如 'model'、'backend'）。
model:
  kerr_cavity:
    chi: 1.0
    epsilon: 2.5

backend:
  numpy:
    float_dtype: float64

integrator:
  euler: {}

    2. 嵌套方案（传统/批量）

    在此方案中，多个任务在单个文件中的 jobs 列表下定义。这对于定义必须一起运行的严格任务序列很有用，但可能更难阅读和维护。

    文件： configs/pipelines/full_pipeline.yaml 

    name: full_pipeline
jobs:
  - name: step_1
    engine:
      sde:
        dt: 0.01
    model:
      kerr_cavity:
        chi: 1.0

  - name: step_2
    input: step_1
    engine:
      viz:
        format: png

    配置层次结构

    为了确定仿真任务的最终设置，QPhase 会从多个来源合并配置。优先级顺序从高到低如下：

    1. 任务配置 (configs/jobs/*.yaml)：特定于单次仿真运行的设置。这些设置会覆盖所有其他设置。
    2. 全局配置 (configs/global.yaml)：项目范围的默认值（例如默认后端、日志偏好）。
    3. 系统默认值：由 QPhase 包及其插件提供的内置默认值。

    任务配置剖析

    任务配置文件定义了仿真的具体参数。

    关键字段

    字段 类型 描述
    name str 必需。 任务的唯一标识符。用于日志和输出文件名。
    engine dict 必需。 仿真引擎的配置。必须恰好包含一个与引擎名称对应的键（例如 sde）。
    model dict 可选。 物理模型插件的配置。
    backend dict 可选。 计算后端插件的配置。
    integrator dict 可选。 积分器插件的配置。
    input str 可选。 指定输入源，例如上游任务的名称。
    output str 可选。 指定输出目标。

    参数扫描

    QPhase 内置支持参数扫描。

    笛卡尔积（网格搜索）

    如果为多个参数提供了列表，QPhase 会为每种可能的组合生成一个任务。

    model:
  kerr_cavity:
    chi: [1.0, 2.0]
    epsilon: [0.1, 0.5, 1.0]

    这将生成 6 个任务（$2 \times 3$）。
