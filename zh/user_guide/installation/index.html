    安装

    本指南介绍 QPhase 及其官方插件的安装过程。

    前提条件

    • 操作系统：Linux、macOS 或 Windows（建议使用 WSL2 以获得更好性能）。
    • Python：3.10 或更高版本。
    • Git：用于克隆仓库。

    推荐：虚拟环境

    我们强烈建议使用虚拟环境，以避免与其他 Python 包发生冲突。

    使用 Conda（推荐）

    conda create -n qphase python=3.11
conda activate qphase

    使用 venv

    python -m venv .venv
# Linux/macOS
source .venv/bin/activate
# Windows
.venv\Scripts\activate

    从源码安装

    目前，QPhase 直接从源代码安装。

    1. 克隆仓库

      git clone https://github.com/PolarisMegrez/qphase.git
cd qphase

    2. 安装核心包

      以可编辑模式 (-e) 安装核心框架，这样您可以修改代码而无需重新安装。

      pip install -e packages/qphase

      对于大多数用户，我们建议安装带有 standard 可选依赖的版本，其中包含常用的包：

      pip install -e packages/qphase[standard]

      这包括 Numba（用于 JIT 编译）和 PyTorch（用于 GPU 加速）。

    3. 安装官方插件

      QPhase 是模块化的。您可以只安装需要的组件。

      • SDE 引擎（随机微分方程）： 

        pip install -e packages/qphase_sde

      • 可视化工具： 

        pip install -e packages/qphase_viz

    4. 安装依赖项

      如果根目录有 requirements.txt 文件，您可以安装它，但通常上述包安装会自动处理依赖。

      # 可选：安装开发依赖（测试、文档）
pip install -r requirements.txt

    验证

    要验证安装是否成功，请运行以下命令：

    qps --version

    您应该看到已安装的 QPhase 核心版本号。

    要检查检测到的插件：

    qps list

    这将显示已注册的后端、引擎和模型列表。

    故障排除

    "Command not found: qps"

    确保您的 Python 环境的 bin（或 Windows 上的 Scripts）目录在系统的 PATH 中。如果您在虚拟环境中安装，请确保已激活该环境。

    "ModuleNotFoundError: No module named 'qphase'"

    这通常发生在安装了依赖但忘记安装包本身的情况下。请再次运行 pip install -e packages/qphase。
